Detailed Description

Abstract class supporting textual IO.

The derived class has to implement a get() and a set() method.

Then this class uses the get() method of derived class as data input. The input is bufferd in a string until input buffer is full (inBufSize) or a received character matches one of the characters in the eos string. The string is complete also if the first reading matches a character in the key string or it is detected as ESC-sequence or as a function key code. Key codes (e.g. arrow key) and ESC sequences are konverted to a single byte (= 0x80|keyCode). The first call to getString() after an input is complete will return a pointer to the input string.

The printf() method is conform to the ANSI-C function and uses the set() method of derived class. The length of the output string is limited to outBufSize.

char * cDevTextIO::getString ( void  )

Read and interprete input data

Each call to this function will initiate a reading from the hardware as far as new data is available. ESC sequences or function key codes are converted:

Key Code Output
up 0x41 0xC1
down 0x42 0xC2
right 0x43 0xC3
